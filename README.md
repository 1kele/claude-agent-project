# Claude Agent Project

Веб-чат интерфейс для работы с Claude Code агентом. Пользователи задают вопросы по технической документации проектов, Claude Code читает файлы и отвечает на их основе.

## Требования

- Docker
- Аккаунт Claude (Pro/Max/Team)

## Запуск

### Через Docker Compose

1. При необходимости создай `.env` из примера:

```bash
cp .env.example .env
```

Для локального запуска значения по умолчанию уже прописаны в `docker-compose.yml`. Для нормального деплоя обязательно поменяй `SECRET_KEY` и `DB_PASS`.

2. Запусти приложение и базу:

```bash
docker compose up --build
```

При старте контейнер приложения сам выполнит:

```bash
alembic upgrade head
```

Первого админа, проекты и доступы нужно создать в базе данных вручную.

### Первый запуск для теста

1. Открой приложение:

```text
http://localhost:8000
```

2. Зарегистрируй первого пользователя через форму в браузере. Для учебного запуска можно использовать:

```text
login: admin
password: 123
```

После регистрации пользователь создаётся как обычный `user`, поэтому кнопку админ-панели он ещё не увидит.

3. Сделай этого пользователя админом в базе:

```bash
docker compose exec postgres psql -U guest -d claudeagent
```

Внутри `psql` выполни:

```sql
UPDATE "user" SET role = 'admin' WHERE login = 'admin';
```

Выйти из `psql`:

```sql
\q
```

4. В браузере нажми `Выйти`, затем снова войди под `admin / 123`. После входа должна появиться кнопка `Админ`.

5. В админ-панели добавь проекты. Названия должны совпадать с папками внутри `projects/`, например:

```text
project_1
project_2
```

6. Чтобы обычный пользователь смог выбрать проект в чате:

- зарегистрируй этого пользователя через форму;
- зайди админом;
- открой `Админ`;
- найди пользователя или введи его ID;
- выбери проекты;
- нажми `Выдать доступ`.

После этого пользователь при входе увидит только те проекты, которые ему выдал админ.

## Авторизация

После первого запуска нужно залогиниться в Claude Code внутри контейнера:

```bash
docker run -it --rm \
  -v ~/.claude:/root/.claude \
  -v ~/.claude.json:/root/.claude.json \
  claude-agent claude auth login
```

Откроется ссылка — перейди по ней в браузере и залогинься через аккаунт Claude. После этого авторизация сохранится и при обычном запуске контейнера уже будет работать.

**Важно:** монтируй папки `~/.claude` и `~/.claude.json` при каждом запуске контейнера, иначе авторизация не найдётся.

2. Запусти контейнер:
```bash
docker run -p 8000:8000 \
  -v ~/.claude:/root/.claude \
  -v ~/.claude.json:/root/.claude.json \
  claude-agent
```

3. Открой браузер: `http://localhost:8000`

## Структура проектов

Документация для проектов лежит в папке `projects/`:
- `projects/project_1/` — первый проект
- `projects/project_2/` — второй проект

## Разработка (без Docker)

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn src.main:app --reload
```
